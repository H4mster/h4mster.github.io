1 Introduction

文本检测在以下方面是高度复杂的:

  1. 复杂的背景
    场景文本可能会出现在多种背景中，例如标志、墙、玻璃之后，从不同的背景中识别出文本并不是一件简单的事。
  2. 多变的文本
    与文档扫描对比，自然场景文本检测要难得多，因为在文档扫描时，文字的形状、颜色、字体、大小、方向几乎都是相同的，并且文档中的文本通常是清晰和水平或垂直对齐的，而在场景文本检测中，除了上述的特征不同外，文字甚至会被其他东西装饰。
  3. 干扰
    文本有着类似的形状，只在细节上有所不同。有些字符有着相似的外型，环境噪声甚至会将一个字符与其他字符混淆。因此，从环境干扰中提取文字是很敏感的，会收到很多外界环境的干扰，例如光照程度，污点，低像素和遮挡。
  4. 文本是一种特殊的对象类型
    文本通常有不同的长宽比，不同的方向。甚至不规则的形状，也就是弯曲的文本。此外，由于识别步骤取决于检测到的文本区域的质量，因此检测模块需要精确地提取与文本区域相同的区域。调整长宽比本身就是一个挑战。不同朝向的文本，甚至弯曲的文本所需要的精确预测也是困难的。

这些困难贯穿于深度学习在CV以及其他方面展现出巨大潜力之前的时期，在AlexNet一举夺得ILSVRC2012的冠军后，研究人员转向通过深度学习模型来自动提取特征，并且开始更深入的研究。近年来取得的进步大致可以归为以下几个方面:

  1. 深度学习带来的好处
    深度学习减去了手动提取特征的繁琐步骤，并且使得算法相较于从前有了很大的提升，并且还产生了端到端的可训练方法，进一步简化了传统的”检测->识别”模式
  2. 面向目标的算法和数据集
    现在的研究转向更加特定的方向。针对现实中不同情况，最新的公共数据集具有更加特别的特点，例如有些数据集根据长文本、模糊文本、弯曲文本进行分类。而在这些数据集的驱动下，几乎所有近年来推出的算法都是为了特定的挑战而设计的，例如，有人提出了面向文本的检测方法，而另一些则针对模糊和失焦的场景图像，这些特殊的想法也被结合在一起，形成更为通用的方法。
  3. 辅助技术的进展
    除了用于主要任务(文本的提取与检测)的新数据集和新模型外，不直接解决任务的辅助技术也在这个领域找到了属于自己的一席之地，例如”合成数据集”,”bootstrapping”(???)等。

在这篇文章中，将会概述文本检测和识别领域的最新发展，重点介绍深度学习时代部分。并且从不同的角度回顾这些方法，列出最新的数据集。分析研究现状，并对未来的研究趋势进行展望。在Section2中，将会简略回顾深度学习时代前的方法。在Section3中，我们将会讨论深度学习技术并且介绍与文本检测与识别有关的算法。在Section4中，我们将会总结并按等级列出基于深度学习的算法。在Section5中，我们将会关注数据集以及评估方法。最后我们列举了一些新开发的应用以及我们对现状和未来趋势的看法。

3 Development of Deep Learning

近年来，深度学习的迅速兴起最终彻底改变了人工智能行业，包括文本检测和识别。深度学习是一组学习算法，它通过自动学习从原始输入中提取特征并匹配输出标签来逼近给定的映射函数。深度学习模型通常包括一系列完全可微的计算步骤，所以将梯度下降方法应用于适当的训练目标中，可以对整个模型进行端到端优化。

深度学习在人工智能的许多领域得到了广泛的应用，与传统的机器学习方法相比有了很大的改进。在本节中，我们将简要介绍与文本检测和识别密切相关和基本的模型。

3.1 Image Classification Task

给定图片以及候选类别以此预测图片，基于深度学习的算法已经超过了传统的机器学习算法甚至超过了人类，而ResNet是第一个性能超过人类的算法。图像分类的进展为其他CV任务奠定了坚实的基础，因为这些任务中的模型通常利用图像分类工作中现成的模型，这些模型被称作base-net,backbone network,stem-network。图像分类任务展现了端到端学习的可能性，这种学习可以在多种任务间共享。

3.2 Object Detection Task

对象检测的目的是从输入图像中检测给定类集合的对象，即定位和识别对象。主要有两个分支，即region-proposal based(基于区域提案的方法)和anchor-based(基于锚的方法?)。这两个分支都间接地启发了基于它们的文本检测算法。

3.2.1 Region-proposal based
R-CNN

The Region-based CNN(R-cnn)提取一个可管理的候选区域，并使用图像分类模型来预测它是否是一个语义对象以及它的类别。

Fast-RCNN

FAST-rcnn将区域建议应用于提取的特征图(feature maps)而不是原始图像，从而加快了流水线的速度(accelerates the pipeline?)，以避免重复计算。

Faster-RCNN

Faster-RCNN包括两个阶段。第一阶段，区域提案网络Region Proposal Network(RPN)生成候选框(candidate object bounding boxes)。第二阶段类似于快速rcnn。

3.2.2 Anchor based

Anchor-based 方法仅一次就能给出预测，Single Shot Detector(SSG)和You-Only-Look-Once(YOLO)算法使用了类似的结构。将图像传递到CNN序列(a sequence of CNNs)后，最终输出为特征映射，其中每个位置(x，y)是表示输入图像中相应区域的特征向量。将分类器和位置回归(position-regression)方法应用于特征向量，预测是否存在语义对象、类和精确位置。

3.3 Semantic Segmentation Task(语义分割任务)

语义分割是一项类似于目标检测的任务，我们需要预测原始图像中每个像素的类别。全卷积网络(Fully Convolutional Network FCN)就是为了完成这项任务而提出的。它由卷积层和池层交替组成，然后是反卷积层，这样设计可以使输出层的大小与输入图像相同。输出层是一个feature map。每个位置的特征向量被输入到分类器中，该分类器预测像素的类别。反卷积层实质上是一种改进的CNN，其中的正向传播和反向传播被交换。因此，输出feature map可以大于输入feature map，即上采样。随后的工作介绍了金字塔结构，上采样侧添加来自下采样部分的特征图，以恢复较低级别的特征。金字塔结构连接的结合非常重要，因为准确的像素级预测需要更多的局部特征。这类技术已广泛应用于文本检测和识别模型。

3.4 Sequence Modeling(序列建模)

序列建模是自然语言处理中的一项重要任务。在文本检测和识别中，由于目标本身就是字符序列，需要考虑序列建模方法。以前的方法使用CRF或基于规则的匹配，而基于深度学习的方法，包括序列对序列（Seq2Seq）学习和基于注意力的Seq2Seq，并在机器翻译任务中取得了显著的改进。Seq2Seq使用了encoder-decoder结构。首先通过字嵌入方法(word embedding method)将输入序列转化为词向量序列。其中encoder是读取输入序列的LSTM，encoder的最后一个隐藏状态用于初始化decoder，而decoder同样是LSTM。decoder不断生成输出序列直至遇到终止符号。
We also refer readers to the following papers for recent advances in machine translation: Transformer [148], Convolutional Seq2Seq [31], [32], and the architecture evaluation survey [11].
这些序列建模模块允许基于端到端梯度学习的文本识别算法.

3.5 Initial Attempts in Text Detection and Recognition(文本检测与识别的初步尝试)

事实上，几十年前，深度学习已经被用于类似的任务。LeNet-5被用在MNIST手写数字识别任务上，错误率低于1%，展现了深度学习在计算机视觉任务上的潜能。

4 Methodology in the Deep Learning Era(深度学习时代的方法论)

近年来的方法有以下两个特点：(1)大多数方法采用基于深度学习的模型;(2)大多数研究者从不同的角度来研究这一问题。基于深度学习的方法能够自动学习特征从而可以避免设计和测试大量潜在手注特征。与此同时，拥有不同观点的研究者们也在丰富和推动着社区更深入的工作,致力于不同的目标，faster and simpler pipeline(??)，text of varying aspect ratios(??)，以及合成数据。正如我们在本节中所看到的那样，深度学习的结合完全改变了研究人员处理这项任务的方式，到目前为止，已经扩大了研究范围。与前一个时代相比，这是最重要的变化。

简而言之，近几年来，对可细分趋势的研究蓬勃发展。在下图中总结了这些变化和趋势，我们将在我们的文章中遵循这张图。

4.1 Detection

在文本检测领域有三个主要趋势，我们将在下面的小节逐一介绍它们。它们是：(1)pipeline simplification(2)changes in prediction units；(3)specified targets

4.1.1 Pipeline Simplification

在深度学习时代之前的大多数方法，以及一些早期使用深度学习的方法，都有multi-step pipelines。最近的方法已经简化并且缩短了pipelines，这是减少错误传播和简化训练过程的关键。然而，这些方法的主要组成部分都是端到端可微模块，即深度学习模型，这是一个突出的特点。

Multi-step methods

早期的基于深度学习的方法将文本检测的任务转化为一个多步骤的过程。例如在《Scene text detection via holistic, multi-channelprediction》中，使用卷积神经网络来预测输入图像(1)中的每个像素是否属于一个字符，(2)是否在文本区域内，(3)像素周围的文本方向，连接的正响应被认为是对字符或文本区域的检测。对于属于同一文本区域的字符，用Delaunay三角剖分，然后根据预测的方向属性将字符划分为文本行。
类似地，在《Multi-oriented text detection with fully convolutional networks》中，首先，预测一个表示哪些像素位于文本行区域内的密集映射。对于每个文本行区域，MSER(最大稳定极值区域算法)用于提取候选字符。候选字符显示出底层文本行的大小和方向的信息。作为最后一步，提取最小边界框作为最终的候选文本行。

Simplified pipeline

更多最近的方法都遵循两步的pipeline，包含一个端到端的可训练神经网络模型和后处理(post-processing)步骤，这通常比以前的要简化得多。这些方法主要是从一般目标检测技术中得到启示，以及得益于高度集成的神经网络模块，可以直接预测文本实例。
主要有两个分支：(1)Anchor-based method,预测文本的存在，仅在输入图像的预定义网格点处返回位置偏移量;(2)Region proposal methods，在提取的图像区域的基础上进行预测和回归。
由于这些工程的最初目标并不仅仅是对pipeline的简化，这里我们只介绍一些有代表性的方法。其他方法将在以下部分介绍。

Anchor-based methods

从SSD中得到灵感，SSG是一种通用的目标检测网络，具体的思想看论文，主要讲了基于标准锚的默认盒预测方法的一个变体——————East。

Region proposal methods

通常遵循r-cnn的标准目标检测框架，其中应用了一种简单而快速的预处理方法，提取一组可能包含文本行的proposal区域。然后，神经网络将其分类为文本/非文本，并通过回归边界偏移来纠正定位。
Rotation Region Proposal Networks 遵循和适应标准的Faster-RCNN框架。为了适应文本的任意方向,RRPN生成旋转可选区域而不是标准的轴对齐矩形。
类似的，R2CNN对基于标准可选区域的目标检测方法进行了修改，以此适应文本的不同比例，选出三个不同规模的兴趣池(Interests Poolings)，并将它们连接起来以进一步预测和回归。
在FEN中，自适应加权池被应用于集成的不同池大小。4个不同大小的poolings被用于计算文本分数。最后利用4个分数来进行预测。

4.1.2 Different Prediction Units

文本检测和一般对象检测的一个主要区别是文本作为一个整体是同构的，并且显示局部性，而一般的目标检测不是。通过同质性(homogeneity)和局部性(locality)，我们提出实例文本的任何部分仍然是文本的属性。人类不需要看到整个文本实例就可以发现图像是文本实例。
这样的属性为文本检测方法的新分支奠定了基础，该分支仅预测子文本组件，然后将它们组装到文本实例中。
在这一部分中，我们从文本检测的细节程度的角度出发。预测细节程度有两个主要级别：文本实例级(text instance level)和子文本级(sub-text level)。
在text instance level方法中:文本的检测遵循一般对象检测的标准例程，其中区域建议网络(region-proposal network)和细化网络(refinement network)结合起来进行预测。区域建议网络产生对可能的文本实例本地化的初始和粗略猜测，然后细化部分将建议区分为文本/非文本，并且还校正文本定位。
在sub-textlevel level方法中:只预测组合成文本实例的部分。这样的子文本主要包括像素级(pixel-level)和组件级(components-level)。
-在pixel-level方法中，端到端全卷积神经网络学习生成一个密集的feature map，指出原始图像中的每个像素是否属于某个文本实例。然后后处理方法把属于同一个文本实例的像素分组在一起。因为文本可以出现在簇(clusters)中，这使得预测的像素相互连接。像素级方法的核心是将文本实例彼此分离。
-在components-level方法中，通常以中等程度的细节进行预测。components引用文本实例的局部区域，有时包含一个或多个字符。
