0x00 序

结合前两篇博客的内容，以猫狗二分类来做一个实战。程序将要实现以下几个功能:

  • 模型定义
  • 数据处理和加载
  • 训练模型(Train & Validate)
  • 训练过程的可视化
  • 测试(Test/Inference)

除了实现功能外，程序还应该满足以下几个要求: 模型需要具有高度可配置性，便于调整参数、修改模型和反复 实验；代码应具有良好的组织结构；代码应具有良好的说明，使他人能够理解。

0x01 文件组织架构

程序最重要的三个功能如下:

  • 模型定义
  • 数据加载
  • 训练和测试

下面是程序文件的组织结构:

各文件的主要内容和作用如下

  • checkpoints/: 用于保存训练好的模型，可使程序在异常退出后仍然能够重新载入模型，恢复训练。
  • data/: 数据相关操作，包括数据预处理、dataset实现等
  • models/: 模型定义，可以有多个模型，例如上面的AlexNet和ResNet34，一个模型对应一个文件
  • utils/: 可能用到的工具函数，本次实验主要封装了一些可视化工具
  • config.py: 配置文件，所有可配置的变量都集中在这个文件中，并提供默认值
  • main.py: 主文件，训练和测试程序的人口，可以通过不同的命令来指定不同的操作和参数
  • requirements.txt: 程序依赖的第三方库

0x02 关于__init__.py

可以看到，几乎每个文件夹下都有一个__init__.py, 一个目录如果包含了__init__.py文件，那么它就变成了一个package。__init__.py可以为空，也可以定义包的属性和方法，但是必须存在，其他程序才能从这个目录导入相应的模块或是函数。例如在data/文件夹下有__init__.py，则在main.py中就可以使用from data.dataset import DogCat，如果在__init__.py中写入from .dataset import DogCat，则在main.py中就可以直接写为: from data import DogCat，或是import data;dataset = data.DogCat，比写成from data.dataset import DogCat更加便捷
